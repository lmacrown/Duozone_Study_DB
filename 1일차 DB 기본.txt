데이터베이스 : 데이터와 베이스의 합성어. 특정 목적을 위해 여러 사람이 공유하여 사용하는 데이터 집합
DBMS : DB 관리 시스템
데이터 : 정제하지 않은 값이나 사실 또는 자료 자체
정보 : 어떠한 목적들을 휘배 데이터를 분석하거나 가공함
효율적인 데이터 관리 조건
1) 데이터를 통합하여 관리
2) 일관된 방법으로 관리
3) 데이터 누락 및 중복 제거
4) 여러 사용자가 공동으로 실시간 사용 가능
DBMS의 장점 : 데이터 중복 제어, 데이터를 동시에 공유, 데이터를 관리하는 방식이 통합됨, 
응용프로그램의 업데이트 또는 변경과 관계없이 데이터를 사용
데이터 모델 : 데이터를 저장하는 방식을 정의해 놓은 개념 모형
- 계층형 데이터 모델 : 트리 구조를 활용. 부모 자식 같은 관계를 정의하고 데이터를 관리. 1대 N 관계
하나의 부모개체가 여러 자식을 가질 수 있지만 자식은 하나만 가질수 있음
- 네트워크형 데이터 모델 : 그래프 구조를 기반. 자식 개체와 부모 객체를 여러 개를 가질 수 있음. M대 N 관계
- 관계형 데이터 모델 : 데이터 간의 관계에 초점을 맞춤
데이터는 별개의 릴레이션으로 정의한 후 사원 정보에 소속된 부서를 식별하는 코드를 부여
개체 : 데이터베이스에서 데이터화하려는 사물. 개념의 정보 단위. 테이블 개념과 대응되며 테이블은 릴레이션으로 표기(객체)
속성 : 개체를 구성하는 데이터의 가장 작은 논리적 단위. 데이터의 종류, 특성, 상태 등을 정의(필드)
관계 : 관계의 연관성을 나타내기 위해 사용. 테이블 간의 관계를 외래키
- 객체 지향형 데이터 모델 : 객체 개념을 기반으로 한 데이터 모델.

관계형 데이터베이스 : 관계형 데이터 모델 개념을 바탕으로 데이터를 저장, 관리하는 데이터 베이스
SQL(Structured Query Language) : 데이터를 다루고 관리하는 데 사용하는 데이터베이스 질의 언어
- DQL(Data Query Language) : RDBMS에 저장한 데이터를 원하느 방식으로 조회하는 명령어
- DML(Data Manupulation Language) : RDBMS 내 테이블의 데이터를 저장, 수정, 삭제 하는 명령어
- DDL(Data Definition Language) : RDBMS 내 데이터 관리를 위해 테이블을 포함한 여러 객체를 생성, 수정, 삭제하는 명령어
- TCL(Transaction Control Language) : 트랜잭션 데이터의 영구 저장, 취소 등과 관련된 명령어
- DCL(Data Control Language) : 데이터 사용 권한과 관련된 명령어 

관계형 데이터베이스의 구성 요소
- 테이블 : 표 형태의 데이터 저장 공간
행 : 하나의 개체를 구성하려는 여러 값
열 : 저장하는 데이터를 대표하는 이름과 공통 특성
키 : 하나의 테이블을 구성하는 여러 열 중에서 특별한 의미를 지닌 하나 또는 여러 열의 조합
- CK(후보키) : 한 테이블에서 각 튜플을 구분하는 유일성과 최소성을 만족하는 데이터.
- PK(기본키) : 유일한 값으로 NULL값과 중복을 허용되지 않음
- AK(대체키) : 후보키가 2개 이상일 떄 기본키를 제외한 나머지 후보키
- SK(슈퍼키) : 유일성은 만족하나 최소성은 만족하지 않는 한 릴레이션 내의 속성들의 집합
- FK(외래키) : 다른 테이블의 기본키를 참조하는 열 데이터. 데이터의 중복 제거
- 복합키 : 여러 열을 조합하여 기본키 여할을 할 수 있게 만든 키

오라클 데이터베이스 : 오라클 사가 만든 DBMS 제품
- 자료형
1) VARCHAR2(길이) : 4000byte만큼의 가변 길이 문자열 데이터를 저장
2) NUMBER : 38자리의 숫자를 저장
3) DATE : 날짜 형식을 저장하기 위해 사용. 초 까지 저장 가능
4) CHAR(길이) : 4000byte만큼의 고정 길이 문자열 데이터를 저장(남은 공간은 공백으로 채움)
5) NVARCHAR2(길이) : 4000byte 만큼의 가변 길이 국가별 문자 데이터 저장
6) BLOB(Binary Long) : 최대 크기 4GB의 대용량 이진 데이터 저장
7) CLOB(Char Long) : 최대 크기 4GB의 대용량 텍스트 데이터 저장
8) BFILE : 최대 크기 4GB의 대용량 이진 데이터 파일을 저장
- 객체 : 데이터를 저장하고 관리하기 위한 논리 구조를 가진 구성 요소
1) 테이블 : 데이터를 저장하는 장소
2) 인덱스 : 테이블의 검색 효율을 높이기 위해 사용
3) 뷰 : 데이터를 논리적으로 연결하여 하나의 테이블처럼 사용
4) 시퀀스 : 일련 번호를 생성해 줌
5) 시노님 : 오라클 객체의 별칭을 지정함
6) 프로시저 : 프로그래밍 연산 및 기능 수행이 가능함
7) 함수 : 프로그래밍 연산 및 기능 수행이 가능
8) 패키지 : 관련 있는 프로시저와 함수를 보관
9) 트리거 : 데이터 관련 작업의 연결 및 방지 관련 기능을 제공

PL/SQL : DB에서 데이터를 관리 하기 위해 별도의 프로그래밍 언어
