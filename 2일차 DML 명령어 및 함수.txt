2. TNS 사용
- tnsnames.ora 파일에 설정

서버 연결 2종류
1. 로컬 연결(기본정보, TNS)
2. 외부 연결(기본정보, TNS)

desc () : 열에 있는 정보를 텍스트로 보여줌
-- : 주석
프로젝션 : 조회할 테이블의 세로 영역, 즉 출력 테이블의 열을 선택하는 방식을 의미
조인 : 두 개 이사으이 테이블을 옆으로 연결하여 하나의 테이블처럼 데이터를 조회하는 방식
SELECT : 데이터베이스에 보관되어 있는 데이터를 조회
FROM : 조회할 데이터가 저장된 테이블 이름을 명시
DISTINCT : 중복데이터 제거
ALL : 모든 데이터를 보여줌. 생략가능
AS : 조회하는 데이터의 별명을 입력해줌. 생략가능
NVL : NULL값을 해당 데이터로 변경
ORDER BY : 데이터를 순서에 맞게 정렬하여 출력. 값이 같을경우 ","를 사용하여 한번 더 정렬 가능
해당 데이터의 별명이나 열의 위치를 써도 가능. 속도 차이가 있으므로 가능하면 사용 안하는 것이 좋음
DESC : 데이터 순서를 역으로 바꿈
AES : 기본순서로 데이터 정렬
WHERE : 데이터의 조건 설정
- AND : 여러 개의 조건이 모두 해당되면 검색
- OR : 여러 개의 조건중 하나만 해당되면 검색
- in() : ()안에 값중 하나만 해당되면 검색 
- NOT in() : () 안에 값이 모두 일치하지 않으면 검색
- != : 값이 같지 않다
- >'F' : 데이터의 첫 문자라 F이상인것만 검색
- BETWEEN : 두 값 사이에 해당되면 검색
- LIKE : 일부 문자열이 포함된 데이터 검색. _는 어떤값이든 상관없이 한개의 문자 의미.
- ESCAPE : 지정한 글자에 해당하는 글자 검색
ex. like 'A\_A%' ESCAPE '\'; => A_AMS
NULL : 데이터베이스에서 중요한 의미가 있는 특수한 데이터 형식
값이 존재하지 않음, 해당 사항 없음, 노출할 수 없는 값, 확정되지 않은 값
색인 생성 시 null 값은 제외
UNION : 두 연산의 결과를 합침(조건이 같아야함)
UNION ALL : 사용시 두 결과가 같아도 중복 출력
INTERSECT : 두 연산의 교집합을 출력
MINUS : 두 연산의 차집합을 출력

문자 함수
UPPER : 문자 데이터를 모두 대문자로 변환
LOWER : 문자 데이터를 모두 소문자로 변환
INITCAP : 문자 데이터 중 첫 글자는 대문자 나머지는 소문자로 변환
검색 : 색인 필드가 함수의 인자로 전달되면 색인을 사용할 수 없음
트리거 : 이벤트 발생시 자동으로 호출됨
LENGTH() : 해당 데이터의 문자길이 출력
SUBSTR(문자열 데이터, 시작위치, 추출 길이) : 데이터를 시작위치부터 길이까지 추출. 시작위치에 - 값도 입력가능
INSTR : 문자열 데이터 안에 문자가 포함되어 있는지 확인
REPLACE : 특정 문자열 데이터에 포함된 문자를 다른 문자로 대체
LPAD/RPAD : 데이터와 자릿수를 지정한 후 남은 데이터는 특정 문자로 채움
CONCAT : 두 문자열 데이터를 합침, ||도 가능
TRIM : 특정문자를 지움(LTRIM, RTRIM)

DESC dual; : dual에 관한 테이블 정보 출력
SELECT 'Hello World' FROM dual; : dual이라는 가상의 테이블을 이용하여 메시지 출력
