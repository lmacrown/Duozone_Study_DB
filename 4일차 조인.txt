Join : 두 개 이상의 테이블을 연결
두 테이블을 동시에 검색했을 경우 모든 경우의 수로 검색이 됨(데카르트 곱)
등가 조인 : 각 테이블의 특정 열에 일치한 데이터를 기준으로 선정
WHERE절의 조건식을 사용해 테이블을 조인할 때 조건식이 테이블 수보다 하나 적음
비등가 조인 : 등가 조인 방식 외의 방식
자체 조인 : 자기 자신과의 테이블과 조인하는 방식. 
하나의 테이블을 여러 개의 테이블처럼 활용하여 조인하는 방식
(+) : 해당 데이터가 일치하지 않아도 값이 출력됨
ex. where e.mgr(+) = m.empno; // mgr값 모두 출력
Outer Join : 두 테이블을 비교해 매칭시켜 매칭되지 않을 경우 NULL값을 반환(Left/Right)
Full Outer Join : Left/Right outer join을 합친 방법
Natural Join : 이름과 같은 열을 찾은 후 그 열을 기준으로 등가 조인
Inner Join : 서로 연관된 내용만 검색하는 조인 방법
Join Using : 공통필드를 명시해주어 조인 기준으로 사용하여 열을 명시
Join on : 조건식을 On에 명시
Cross Join : 두 테이블에서 곱집합을 반환하는 방법

서브쿼리 : SQL문을 실행하는 데 필요한 데이터를 추가로 조회하기 위해 SAL문 내부에서 사용하는 select문을 의미
1) 비교 또는 조회 대상의 오른쪽에 놓이며 괄호로 묶어서 사용
2) order by를 사용할 수 없음
3) 서브쿼리의 결과가 메인쿼리의 연산자 종류와 호환가능해야 함
4) 비교 대상과 같은 개수와 같은 자료형으로 지정해야 함
단일행 서브쿼리 : 실행 결과 행이 한개
다중행 서브쿼리 : 실행 결과 행이 여러 개
- in : 메인쿼리의 데이터가 서브쿼리의 결과 중 하나라도 일치하면 true(any와 같음)
- any : 메인쿼리의 조건식을 만족하는 서브쿼리의 결과가 하나 이상이면 true(안쓰는게 좋음)
- all : 모든 데이터가 만족해야 됨
- exist : 서브쿼리의 결과가 하나라도 존재하면 true
exist와 in 의 차이 : in은 값이 일치해도 모든 데이터를 매칭하나 exist는 하나라도 일치하면 실행종료함

table : 물리적
view : 논리적. 서브커리를 인라인 뷰라고 부름
with절 : 가독성이나 성능을 보완하기 위해 사용
