pivot : 기존 테이블 행을 열로 바꿈
unpivot : 기존 테이블 열을 행으로 바꿈

테이블 생성 : create table (테이블 명) as select (가져올 데이터)
where 1<> 1을 사용할 경우 테이블에 데이터가 없이 생성됨
ex. create table emp_temp as select * from emp where 1 <>1;
insert into로 값을 대입할때 빈 값은 null로 들어감
서브쿼리로사용하여 한번에 여러 데이터를 추가할 때 주의점
1) values절을 사용하지 않음
2) 추가되는 테이블의 열 개수와 서브쿼리의 열 개수가 일치
3) 테이블의 자료형과 서브쿼리의 자료형이 일치
데이터 수정 : update (변경할 테이블) set (컬럼) where(변경할 대상 조건)
조건을 지정하지 않을 경우 모든 데이터가 변경됨
ROLLBACK : 테이블 내용을 이전 상태로 되돌림

트랜잭션 : 데이터베이스의 상태를 변화시키는 작업의 단위
취소 : rollback
저장 : commit

세션 : 서버의 시작부터 끝을 의미. 데이터 베이스 접속을 시작으로 접속을 종료하기 까지의 전체 기간을 의미
읽기 일관성 : 어떤 특정 세션에서 데이터가 변경 중일 때 다른 세션에서는 확인할 수 없고 본래의 데이터를 보여 줌
Lock : 특정 세션에서 조작중인 데이터는 트랜잭션이 완료되기 전까지 다른 세션에서 조작할 수 없음
접근을 보류시키는 것
- 행 레벨 록 : 행만 LOCK이 발생
- 테이블 레벨 록 : 테이블 단위로 잠금

TCL : 트랜잭션 제어 명령어
DML : 데이터 조작어
DDL : 데이터 정의어. 명령어를 수행시 바로 데이터 베이스에 commit 함으로 롤백할 수 없음

테이블,열 이름 생성 규칙
1. 문자로 시작해야 함. 한글도 가능하며 숫자로는 불가능
2. 이름은 30byte이하
3. 같은 사용자 소유의 이름은 중복될 수 없음
4. 이름은 영문자, 숫자와 특수문자 사용 가능
5. SQL키워드로 이름을 지을수 없다

ALTER : 생성된 오라클 데이터베이스 객체를 변경할 때 사용
rename (변경할 테이블) to (변경할 이름) : 테이블 명 변경
add : 테이블에 새 열을 추가
modify : 특정 열의 자료형이나 길이 변경
drop : 특정 열을 삭제
truncate : 테이블의 모든 데이터를 삭제함. 데이터 정의어이기 때문에 rollback되지 않음
savepoint : 이전까지 작성한 sql문 실행을 저장함

데이터 사전 : 데이터 베이스를 구성하고 운영하는 데 필요한 모든 정보를 저장하는 특수한 테이블. 
데이터베이스가 생성되는 시점에 자동으로 만들어짐
- user_xxxx : 현재 데이터베이스에 접속한 사용자가 소유한 객체 정보
- all_xxxx : 사용자가 소유한 객체 중 사용 허가를 받은 객체. 사용가능한 모든 객체 정보
- dba_xxxx : 데이터베이스를 관리를 위한 정보
- v$_xxxx : 데이터베이스 성능 관련 정보

색인 :  데이터 검색 성능의 향상을 위해 테이블 열에 사용하는 객체
인덱스의 효율을 높이기 위해서는 데이터가 잘 분포되어있어야한다.
- 복합 인덱스 : 두 개 이상 열로 만들어지는 인덱스
- 고유 인덱스 : 열에 중복 데이터가 없을 때 사용
- 함수 기반 인덱스 : 열에 산술식 같은 데이터 가공이 진행된 결과로 인덱스 생성
- 비트맵 인덱스 : 데이터 종류가 적고 같은 데이터가 많이 존재할 때 주로 사용

뷰 : 하나 이상의 테이블을 조회하는 SELECT문을 저장한 객체
사용 목적 : 편리성과 보안성
- or replace : 현재 생성한 뷰로 대체하여 생성
- with check option : 조건을 만족하는 데이터에 한해 DML 작업이 가능하도록 뷰 생성
- with read only : 뷰의 열람만 가능

오라클 MVIEW  : 사용자가 MVIEW를 이용할 때 데이터를 가져오고 사용자에게 반환하고 뷰에서 삭제하지 않음
성능향상을 기대하나 원본 테이블과의 데이터 동기화 문제가 발생할 수 있음
테이블끼리 조인이 되거나 통계데이터에서 사용
