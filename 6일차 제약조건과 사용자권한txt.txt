시퀀스 : 오라클 데이터베이스에서 특정 규칙에 맞는 연속 숫자를 생성하는 객체
시퀀스 수정 : alter sequence
- increment by : 해당 숫자만큼 증가
- start with : 해당 숫자에서 시작
- cycle : 반복여부 결정
- cache : 캐시를 생성해줌. 할당 된 캐시수만큼 속도가 빨라짐. 20개가 적당함

동의어 : 객체 이름 대신 사용할 수 있는 다른 이름을 부여. 일회성이 아닌 모든 테이블에서 사용가능
제약조건 : 테이블에 저장할 데이터를 제약하는 특수한 규칙을 의미
- not null : 지정한 열에 null을 허용하지 않음
테이블의 속성을 변경할 때 not null로 변경하려고 할때 데이터가 없으면 오류가 뜸
- unique : 지정한 열이 유일한 값을 가짐(중복안됨)
- primary key : 지정한 열이 유일하면서 null을 허용하지 않음
- foreign key : 다른 테이블의 열을 참조하여 존재하는 값만 입력
서로 다른 테이블간 관계를 정의하는 데 사용
- check : 설정한 조건식을 만족하는 데이터만 입력 가능
데이터 무결성 : 데이터베이스에 저장되는 데이터의 정확성과 일관성을 보장한다는 의미
- 영역 무결성 : 열에 저장되는 값을 정해 놓은 범위를 만족하는 데이터임을 규정
- 개체 무결성 : 기본키는 반드시 값을 가지고 중복과 null을 허용하지 않음
- 참조 무결성 : 외래키 값은 참조 테이블의 기본키로서 존재해야 하며 null이 가능
1) 개체 무결성 : 기본키는 반드시 값을 가지고 유일성을 보장하는 최소 집합
2) 참조 무결성 : 외래키 값은 NULL이거나 참조 테이블의 기본키 값과 동일
3) 도메인 무결성 : 특정 속성 값이 미리 정의된 도메인 범위에 속해야 한다
제약 조건 삭제 : alter 명령어에 drop constraint를 사용
delete cascade : 부모정보가 사라지면 자식에 있는 정보도 삭제됨
delete set null : 부모 정보가 삭제되면 자식 데이터는 null이 됨
constraint : 제약 조건의 이름을 직접 지정
rename constraint ~ to ~: 이미 생성된 제약 조건 이름 변경
constraint~references : 

check 제약 조건 : 열에 저장할 수 있는 값의 범위 또는 패턴 정의
default : 기본값 정의
disable : 제약조건 비활성화
enable : 제약조건 활성화

사용자 : 데이터베이스에 접속하여 데이터를 관리하는 계정
스키마 : 오라클 데이터베이스에 접속한 사용자와 연결된 객체
계정 생성 : create user ~ identified by ~
select * from all _users : 모든 사용자 정보 출력
시스템 권한 : 사용자 생성과 정보 수정 및 데이터베이스 관리 권한이 사용자가 부여할 수 있는 권한
with : grant문을 통해 부여받은 권한을 다른 사용자에게 부여 할 수 있는 권한도 함께 부여
롤 : 여러 권한을 하나의 이름으로 묶어 권한 부여 관련 작업을 간편하게 하기위해 사용
리소스 : 오라클데이터베이스에서 제공하는 롤
사용자를 생성 할 때 사용 테이블 스페이스의 영역을 무제한 사용 가능
쿼터 : 계정에게 한정적으로 권한을 줌

객체 권한 : 특정 사용자가 생성한 권한. 
grant [객체 권한/all privileges] on [스키마, 객체 이름] to [사용자 이름/롤 이름/ public] [with grant option]
객체 권한 취소 : revoke
롤 : 자동적으로 권한을 줌
사용자 정의 롤 : 필요에 의해 직접 권한을 포함시킨 롤
connect by : 자료가 이어진 형태를 트리 형태로 나타냄
